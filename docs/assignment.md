## **The task**

Build a small backend that lets someone browse \~200,000 products (newest first), filter by category, and paginate through them.

1. Pagination should be fast.  
2. Show the correct data while data is changing. If 50 new products are added/updated while someone is browsing, they must not see the same product twice or miss one. 

We are intentionally not telling you exact requirements, working out the right approach, and learning it if it's new to you, is the whole point. Preferably Python or NodeJs. And Choose any DB you find fit.

## **The data**

In the database, generate the products yourself. Write a small script that creates **200,000 products**, each with a name, category, a price, a unique id, updated\_at, created\_at time. It’s fine for many products to share the same column values. Commit this generation/seed script to your repo so we can see how you did it. Tip: don’t do a slow approach in a loop.
